Configurando o Apache e o PHP

  1. Iniciar o Wamp
  2. Abrir o arquivo de configuração do Apache httpd.conf
    Clique no ícone do Wamp -> apache -> httpd.conf
  3. Substituir as linhas:
    DocumentRoot "C:\wamp\www" e <Directory C:\wamp\www\>
    Por:
    DocumentRoot "E:\NetbeansProjects\" e <Directory E:\NetbeansProjects\>
  4. Colocar a pasta do projeto no diretório D:\NetbeansProjects
  5. Habilitar o modo rewrite do Apache
    Clique no ícone do Wamp -> Apache -> Apache modules
    Marque a opção [rewrite_module]
  6. Acesse a URL: localhost/prontuariodigital

Utilização do MVC

Criando a View

Atenção! Toda View deve ser criada na pasta views do projeto

1) Crie um arquivo chamado pessoas.php dentro da pasta view com o seguinte código: 

<?php include_once VIEWS_PATH . 'layouts/default/header.php'; ?>

    <div class="content-wrapper">
        <section class="content">

            <h2>View da página pessoas.php</h2>

        </section>
    </div>

<?php include_once VIEWS_PATH . 'layouts/default/footer.php'; ?>
O que aconteceu:
  1. Na linha 2 incluímos o header do layout
  2. Nas linhas 3 e 4 criamos os dois elementos principais para envelopar (wrap) o conteúdo principal da página
  3. Na linha 6 incluimos o conteúdo da página, no caso, apenas uma tag de título <h2> com um texto dentro
  4. Nas linhas 8 e 9 fechamos as tags do conteúdo principal
  5. E por fim na linha 11 incluímos o footer do layout
Atenção! Toda View precisa de um controller para que este a carregue

Criando a Controller

2) No diretório controller crie um arquivo chamado PessoasController.php com o seguinte código:

<?php
class PessoasController extends Controller
{	
	public function index()
	{
        $this->loadView('pessoas');
	}
}
O que aconteceu:
  1. Na linha 2 criamos a classe (objeto) PessoasController e herdamos a classe base Controller
    Todos os controllers deverão começar com a(s) primeira(s) letras iniciais maiúsculas e ao final juntar com a palavra Controler, também com a inicial em maiúsculo
  2. Na linha 4 criamos o método (função) index()
    Todos os controllers deverão ter um método index(), pois será o método padrão caso não for informado nenhum outro método
  3. Na linha 6 fizemos a chamada do método loadView() da aplicação
  4. Como parâmetro passamos uma string com o nome do arquivo da nossa view pessoas.php mas sem a extensão .php
  5. Este método se encarregará de formatar a string carregar a nossa view pessoas.php
  6. Por fim acesse a URL: localhost/prontuariodigital/pessoas

    public function inserir()

    }

	    // Script para calcular a idade da pessoa de acordo com a data de nascimento
	}

Atenção! Toda Model deve ser criada na pasta models do projeto e seu nome deve seguir o padrão ClasseModel.php e herdar a classe Model principal
O que aconteceu:
  1. Na linha 2 criamos a classe (objeto) PessoasModel e herdamos a classe base Model
    Todas as models, assim como os controllers, deverão começar com a(s) primeira(s) letras iniciais maiúsculas e ao final juntar com a palavra Model, também com a inicial em maiúsculo
  2. Na linha 5 e 6 criamos duas propriedades privadas do objeto Pessoa nome e dataNascimento (a maioria das propriedades serão privadas para evitar o acesso direto à elas)
    Por padrão as propriedades devem começar com a letra mínuscula da primeira palavra e as demais com maiúscula
  3. Na linha 8 criamos (sobreescrevemos) o nosso construtor padrão para iniciarmos algumas variáveis que serão úteis para trabalharmos com a model
  4. Da linha 26 té a 44 criamos os métodos getters e setters para manipularmos os dados do objeto enquanto existentes (estes métodos devem ter acesso público para a manipulação dos dados do objeto)
    5. Por padrão os métodos, assim como as propriedades, devem começar com a letra mínuscula da primeira palavra e as demais com maiúscula
  5. Apartir da linha 49 ficará toda a regra de negócio da aplicação onde esta, irá se comunicar com o banco para buscar dados, fazer cálculos para retornar algum resultado, etc (o tipo de acesso dependerá da regra de negócio)
